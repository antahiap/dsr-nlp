11.  Phase-indexed Latent Heat Table.   A table ID may be speciﬁed for the Latent 
heat (LAT1) to describe each phase change individua lly.  Use *DEFINE_TA-
BLE_2D and set the abscissa values to the correspon ding phase transition 
number.  That is, 2 for the Austenite – Ferrite, 3 for the Austenite – Pearlite, 4 
for the Austenite – Bainite and 5 for the Austenite  – Martensite.  See Remark 10  
for an example of a correct table deﬁnition.  If a curve is missing, the corre- 
sponding latent heat for that transition will be se t to zero.  Also, when a table is 
used, LAT2 is ignored.  If HEAT > 0, the latent for  the transition back to Aus- 
tenite can also be included.  This latent heat curv e is marked as 1 in the table 
deﬁnition of LAT1. 
12.  Phase-indexed Thermal Expansion Table.   Tables are supported for deﬁning 
diﬀerent thermal expansion properties for each phas e.  The input is identical to 
the above table deﬁnitions.  The table must have th e abscissa values between 1 
and 5 where the number correspond to phase 1 to 5. To exclude one phase 
from inﬂuencing the thermal expansion you simply in put a curve that is zero 
for that phase or even easier, exclude that phase n umber in the table deﬁnition.  
For example, to exclude the bainite phase you only deﬁne the table with curves 
for the indices 1, 2, 3 and 5.
